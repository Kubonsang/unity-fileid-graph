# v0.9 Transform Reparent Experiment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an experimental `uyaml prefab reparent-transform <file> --id <transformFileID> --new-parent <transformFileID> --experimental --write` command that changes one Transform parent link while preserving graph safety guarantees.

**Architecture:** Keep the `v0.6`/`v0.7` structural-mutation philosophy: no generic YAML serializer, no scene support, and no broad graph rewriting. Reparenting must be implemented as a constrained edit over `Transform.m_Father` plus the affected `m_Children` lists, followed by the existing pre-check, temp-check, backup, replace, and final-check pipeline.

**Tech Stack:** Go stdlib, existing `internal/parser`, `internal/graph`, `internal/check`, `internal/mutate`, `go test`

---

## Scope Lock

- namespace: `prefab` only
- command: `reparent-transform`
- required flags: `--id`, `--new-parent`, `--experimental`, `--write`
- one-child-one-parent move only
- update exactly these relationships:
  - child `Transform.m_Father`
  - old parent `Transform.m_Children`
  - new parent `Transform.m_Children`
- reuse the shared temp-check / backup / replace / final-check / restore pipeline

Out of scope:

- `scene` mutation
- cycle repair by guessing
- moving whole GameObject subtrees by name
- generic YAML editing
- multiple reparent operations in one command

```go
func validateReparent(graphResult *core.Graph, childID, newParentID int64) (string, string) {
	child := graphResult.Transforms[childID]
	if child == nil {
		return core.MutationCodeMissingTransformComponent, "child transform is missing"
	}
	if child.Father == 0 {
		return core.MutationCodeRootReparentBlocked, "reparenting a root transform stays blocked in v0.9"
	}
	if childID == newParentID {
		return core.MutationCodeTransformCycleBlocked, "child transform cannot become its own parent"
	}
	current := newParentID
	for current != 0 {
		if current == childID {
			return core.MutationCodeTransformCycleBlocked, "reparent would introduce a transform cycle"
		}
		node := graphResult.Transforms[current]
		if node == nil {
			break
		}
		current = node.Father
	}
	return "", ""
}
```

```md
- new parent `m_Children` must append exactly one `{fileID: child}`

- temp-check `ERROR` prevents replace
- final-check `ERROR` triggers restore-on-final-check-error just like `remove-component`
```
